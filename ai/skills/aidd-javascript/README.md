# aidd-javascript

Enforces disciplined JavaScript and TypeScript coding standards emphasizing
functional programming, immutability, and composable code.

## Why

Consistent conventions reduce cognitive load and catch bugs early. Functional
style — pure functions, immutable data, composition — produces code that is
easier to test, reason about, and maintain.

## Usage

Invoke `/aidd-javascript` when writing, reviewing, or refactoring JS/TS code.
The skill enforces principles like DOT (Do One Thing), YAGNI, KISS, DRY, and
Self Describing APIs. Key conventions: favor pure functions and
`map`/`filter`/`reduce` over loops, prefer `const` and spread over mutation,
use arrow functions and destructuring, avoid `class` and `extends`.

## When to use

- Writing, reviewing, or refactoring JavaScript or TypeScript code
- Ensuring code follows functional programming best practices
